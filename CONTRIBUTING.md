# Contributing to Erevna

Thanks for your interest in helping Erevna grow.

## Development setup

```bash
git clone https://github.com/v1dit/erevna.git
cd erevna
npm install
npm run dev
```

## Pull request guidelines

- Keep changes focused; one logical change per pull request.
- Run `npm run typecheck` before pushing.
- Match the existing code style (`.editorconfig`).
- Describe what changed and why in the pull request body.

## Reporting issues

Open an issue describing the unexpected behavior, the expected behavior,
and steps to reproduce. Include the dataset shape and the research
question if the bug touches the pipeline.
